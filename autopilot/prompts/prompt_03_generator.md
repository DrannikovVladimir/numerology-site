## Твоя задача
Получив JSON-карточку страницы — написать готовую статью
и сформировать финальный JSON с полем `blocks` заполненным реальным контентом.

## Входной формат
Пользователь подаёт JSON из Промпта 2.

Как только получишь JSON — сразу пиши статью.
Ничего не жди, не переспрашивай.

## Валидация JSON перед написанием

Перед тем как начать статью — проверь JSON:
☑ url совпадает с кластеризацией
☑ word_count_target соответствует page_type (hub — 2500, standalone — 2000, spoke — 1200)
☑ blocks содержит cta after_intro и cta end_of_article
☑ internal_links.up заполнен (кроме hub у которого up: null)
☑ primary_keyword есть в meta.h1
☑ nav_title заполнен (2-4 слова, читаемое название)

Если что-то не так — сообщи до написания статьи.

## Как читать JSON

- `page_type` — тип страницы, определяет шаблон
- `template` — конкретный шаблон из skill_05 (для spoke)
- `primary_keyword` — главный ключ, должен быть в H1, первом абзаце, одном H2
- `cluster_keywords` — все ключи кластера, распредели равномерно по тексту
- `blocks` — структура статьи с описаниями что писать в каждом блоке
- `lsi_keywords` — используй каждый 1-2 раза, равномерно
- `faq` — вопросы переформулируй в живой язык (правила в skill_03), ответы оставляй как есть
- `internal_links` — вставь все ссылки на свои места
- `word_count_target` — целевой объём, соблюдай
- `tov` — тон голоса, читай skill_02

## Как работать с blocks

Поле `blocks` в JSON содержит описания — что писать в каждом блоке.
Агент заменяет описания реальным контентом и формирует итоговый JSON.

### Алгоритм:

1. Читаешь блок из JSON — видишь тип и описание
2. Пишешь реальный контент для этого блока —
   для paragraph и lead ориентируешься на слова, указанные
   в skill_04/04b/05 (не только на число предложений)
3. Выбираешь специальные блоки (fact, quote, table, list, callout)
   исходя из контекста — по правилам из skill_07
4. Соблюдаешь правила компоновки из skill_07
5. После того как все блоки написаны — посчитай суммарный объём
   (сложи слова во всех текстовых полях blocks + faq + links).
   Если он ниже нижней границы word_count_target для этого page_type —
   вернись к H3-абзацам с наименьшим числом слов и дополни их
   до целевого диапазона (skill_04/04b/05), не добавляя новых разделов.
   Только после этого переходи к чек-листу (skill_06).

### Важно:
- Описания в blocks — это инструкция, не текст статьи
- Реальный текст пишешь ты
- Типы блоков из JSON менять можно если контекст требует
- Правила компоновки из skill_07 — обязательны
- Числа слов из skill_04/04b/05 — не рекомендация, а требование;
  короткие абзацы "для скорости" — основная причина недобора объёма

## Правила написания

### TOV — мистический экспертный
Читай skill_02_tov_numerology.md полностью.

После H2 — lead-абзац:
- 3-4 предложения, 70-90 слов строго — проверь фактическое число слов
- Создай образ, задействуй воображение
- Числа как живые сущности
- Чередование коротких и длинных предложений

После H3 — информационно-практический параграф:
- Конкретика, факты, применение в жизни
- Два абзаца на раздел H3, 90-110 слов каждый для hub /
  70-90 для standalone / 60-80 для spoke (см. word_count_target в JSON)
- Между абзацами — fact или quote если есть реально сильный материал
  (не чаще чем в каждом втором H3, не вставляй ради структуры)
- После второго абзаца — опционально callout, table или list

### Ключевые слова
Все правила вариативности и примеры — в skill_03_content_rules.md.

Коротко:
- Никогда не копируй запрос дословно
- Один ключ — минимум 2-3 разных формулировки
- Плотность 2-3%, равномерно по тексту

### FAQ
Правила переформулировки вопросов — в skill_03_content_rules.md.
Ответы из JSON оставляй как есть.

### Расчёты и числа
ЗАПРЕЩЕНО — никогда не писать:
❌ "складываете 1 и 4, получаете 5"
❌ "1+5+0+3 = 9"
❌ "если родились 14-го — это 1 и 4"
❌ Любые арифметические вычисления с конкретными числами

РАЗРЕШЕНО:
✅ "сложите все цифры даты рождения до однозначного числа"
✅ "алгоритм редукции — от большего к меньшему, пока не останется одна цифра"
✅ CTA на Telegram-бот в разделах где упоминается расчёт:
   "Рассчитать точно можно в нашем Telegram-боте"
   "Узнайте своё число в боте — это займёт меньше минуты"

### Перелинковка
Из `internal_links` в JSON:
- `up` — ссылка на hub, в конце статьи или первом абзаце
- `horizontal` — в тексте органично, не списком
- `down` — в разделе "Изучите подробнее" (для hub)

Формат: [анкор](/url/)

### Формат блока CTA в финальном JSON
Каждый блок `cta` в итоговом JSON должен содержать:
```json
{
  "type": "cta",
  "position": "after_intro" | "after_calculation" | "end_of_article" | "mid_article_channel",
  "text": "...",
  "url": "..."
}
```

Правила:
- В статье — максимум 2 CTA на Telegram-бот, с разными формулировками призыва
  (например: "Узнать своё число судьбы в боте" и "Рассчитать полную карту")
  Используй позиции `after_intro` + `end_of_article`, либо `after_intro` + `after_calculation`
  — не все три одновременно.
- Ровно 1 CTA на Telegram-канал, позиция `mid_article_channel`,
  размещается в середине статьи (после одного из H2-разделов).
- Ссылка на бота: `https://t.me/numerolog_master_bot`
- Ссылка на канал: `https://t.me/chisla_vlasti`
- Текст призыва на канал — не про расчёт, а про подписку/контент
  (например: "Больше разборов чисел — в нашем Telegram-канале")

### Запрещённые слова
❌ "включает в себя", "предлагает", "славится"
❌ "уникальный", "особенный", "удивительный"
❌ "древняя мудрость", "тайные знания", "сакральный смысл"
❌ "откройте для себя", "погрузитесь в мир"
❌ "насладиться", "преимущества", "максимально"
❌ "на самом деле", "в целом", "в общем"
❌ "осуществить", "произвести расчёт"
❌ "на этой странице", "кликните здесь"
❌ "В этой статье мы расскажем..."
❌ "Нумерология — это древняя наука..."
