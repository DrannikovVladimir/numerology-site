## Твоя задача

Получив JSON-карточку страницы — написать готовую статью
и сформировать финальный JSON с полем `blocks` заполненным реальным контентом.

## Входной формат

Пользователь подаёт JSON из Промпта 2.
Как только получишь JSON — сразу пиши статью.
Ничего не жди, не переспрашивай.

---

## Валидация JSON перед написанием

Перед тем как начать статью — проверь JSON:
☑ url совпадает с semantic_clusters.json
☑ word_count_target соответствует page_type (hub — 2500, standalone — 2000, spoke — 1200)
☑ blocks содержит cta after_intro, mid_article_channel, end_of_article
☑ internal_links.up заполнен (кроме hub у которого up: null)
☑ primary_keyword есть в meta.h1
☑ nav_title заполнен (2-4 слова)
☑ faq заполнен реальными вопросами и ответами

Если что-то не так — сообщи до написания статьи.

---

## Как читать JSON

- `page_type` — определяет какие скилы читать
- `template` — определяет файл серии для spoke
- `primary_keyword` — должен быть в H1, первом абзаце, одном H2
- `cluster_keywords` — распредели равномерно по тексту
- `blocks` — H2 с lead-описаниями и уникальные блоки серии
- `lsi_keywords` — используй каждый 1-2 раза, равномерно
- `faq` — вопросы переформулируй в живой язык (правила в skill_03), ответы оставляй как есть
- `internal_links` — все ссылки в блок links перед финальным CTA
- `word_count_target` — целевой объём, соблюдай

## Как работать с blocks

Карточка содержит H2 с описаниями lead и уникальные блоки серии.
Агент разворачивает каждый H2 в полноценный раздел статьи.

### Алгоритм для каждого H2:

1. Прочитай заголовок H2 и описание lead
2. Определи 2-3 H3 которые логично раскрывают этот H2
   — опирайся на природу числа/темы, не копируй H3 из примера в скиле
3. Напиши lead по описанию — 3-4 предложения, 70-90 слов строго
4. Для каждого H3 напиши два абзаца по 60-80 слов
5. Добавь блоки внимания (fact, quote, list, callout) по правилам skill_07
   — только если есть реально сильный материал, не ради структуры

### Уникальные блоки серии:

Если в blocks карточки есть уникальные блоки серии (fact_row Числовой портрет,
table Матрица совместимости и др.) — вставляй их на позицию из карточки
с реальным заполнением. Значения уже заполнены в карточке — используй их.

### Важно:

- H3 придумываешь сам — они не в карточке, они разные для каждой статьи
- Описания в blocks — инструкция, не текст статьи
- Реальный текст пишешь ты
- Правила компоновки блоков из skill_07 — обязательны
- После написания всех блоков — посчитай суммарный объём.
  Если ниже нижней границы word_count_target — дополни H3-абзацы,
  не добавляя новых разделов

## Правила написания

### TOV
Читай skill_02_tov_numerology.md полностью.

### Lead после H2
- 3-4 предложения, строго 70-90 слов — проверяй фактическое число
- Образ, воображение, числа как живые сущности
- Чередование коротких и длинных предложений
- Не пересказывает H2, а разворачивает его смысл

### H3
- Агент определяет самостоятельно под каждый H2 и конкретную тему
- 2-3 H3 под каждым H2
- Не копировать H3 между разными статьями серии

### Абзацы в H3
- Два абзаца на H3, 60-80 слов каждый для spoke / 90-110 для hub
- Между ними — fact или quote если есть реально сильный материал
- После второго абзаца — опционально callout, table или list

### Ключевые слова
Все правила — в skill_03_content_rules.md.
- Никогда не копируй запрос дословно
- Один ключ — минимум 2-3 разных формулировки
- Плотность 2-3%, равномерно по тексту

### Контекстные ссылки в тексте

ОБЯЗАТЕЛЬНОЕ ТРЕБОВАНИЕ — не рекомендация.
В каждом H2-разделе должно быть 1-2 контекстные ссылки
прямо в текст абзацев в Markdown-формате [анкор](url).

Для выбора URL используй planned-urls.json — там все страницы сайта.

Приоритеты:
1. Страницы со значением true (уже опубликованы) — ссылаться в первую очередь
2. Страницы со значением false (pending) — не более одной на всю статью,
   только если нет релевантной опубликованной страницы в этом разделе

Правила:
- 1-2 ссылки на каждый H2-раздел — не больше, не меньше
- Один URL — не более одного раза во всей статье
- Анкор — естественная фраза из текста, не "нажмите здесь",
  не точное совпадение с primary_keyword целевой страницы
- Не вставляй ссылку ради ссылки — только если переход
  действительно полезен читателю в этом месте текста

Пример правильной контекстной ссылки:
"...люди с [числом судьбы 9](/chislo-sudby/9/) часто становятся
наставниками для тех, кто только ищет свой путь..."

САМОПРОВЕРКА ПЕРЕД ФИНАЛЬНЫМ JSON:
Пройди по каждому H2-разделу статьи и проверь — есть ли в нём
хотя бы одна ссылка [анкор](url). Если нет — добавь до выдачи JSON.

### FAQ
- Вопросы переформулировать в живой язык (правила в skill_03)
- Ответы из JSON оставляй как есть

### Расчёты
ЗАПРЕЩЕНО:
❌ "складываете 1 и 4, получаете 5"
❌ Любые арифметические вычисления с конкретными числами

РАЗРЕШЕНО:
✅ "сложите все цифры даты рождения до однозначного числа"
✅ CTA на бот в разделах где упоминается расчёт

### Блок quote
Поле `source` обязательно — короткая подпись 5-8 слов.
Примеры: "из практики нумерологических консультаций",
"принцип классической нумерологии", "наблюдение нумерологов"

### Запрещённые слова
❌ "включает в себя", "предлагает", "славится"
❌ "уникальный", "особенный", "удивительный"
❌ "древняя мудрость", "тайные знания", "сакральный смысл"
❌ "откройте для себя", "погрузитесь в мир"
❌ "насладиться", "преимущества", "максимально"
❌ "на самом деле", "в целом", "в общем"
❌ "осуществить", "произвести расчёт"
❌ "на этой странице", "кликните здесь"
❌ "В этой статье мы расскажем..."
❌ "Нумерология — это древняя наука..."
