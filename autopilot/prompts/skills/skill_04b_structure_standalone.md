# skill_04b_structure_standalone.md

## Структура standalone-статьи

### Что такое standalone-страница
Standalone — самостоятельная страница которая:
- Не является hub и не имеет дочерних spoke
- Раскрывает одну конкретную тему целиком
- Ранжируется по среднечастотным запросам
- Связана с ближайшим hub через перелинковку

### Технические параметры

**Объём:** 1500–2000 слов
**Заголовков H2:** 6–7
**Заголовков H3:** 2 под каждым H2
**Списков:** 1-2 на статью
**FAQ:** 2-3 вопроса

### Мета-данные

**Title (до 60 символов):**
- Главный запрос + выгода или интрига
- Без воды, без "лучший", "топ", "№1"
- Пример: "Ведическая нумерология: значение чисел и расчёт"

**Description (до 155 символов):**
- Раскрывает что найдёт пользователь
- Содержит главный запрос
- Лёгкий призыв к действию в конце
- Пример: "Ведическая нумерология по дате рождения: как считать,
  что означают числа. Отличия от Пифагорейской и Халдейской систем."

**H1 (до 60 символов):**
- Близко к главному запросу но живее
- Не копирует title дословно
- Пример: "Ведическая нумерология — числа сквозь призму Вед"

### Как строить структуру standalone

Агент не использует фиксированный шаблон H2.
Структура строится исходя из семантики конкретного кластера.

Для каждой standalone агент задаёт себе вопросы ниже и выбирает те,
которые покрываются запросами кластера. Последний вопрос (FAQ) обязателен всегда.

**Вопросы для построения структуры:**

- Что это такое и в чём суть?
  *Пример H2: "Что такое ведическая нумерология и чем она отличается"*

- Как это работает / из чего состоит?
  *Пример H2: "Три ключевых числа в ведической традиции"*

- Как рассчитать / получить результат?
  *Пример H2: "Как рассчитать ведическую нумерологию по дате рождения"*

- Что означают результаты?
  *Пример H2: "Значение чисел в ведической нумерологии"*

- Как применять на практике?
  *Пример H2: "Как использовать ведическую нумерологию в жизни"*

- Чем отличается от похожих инструментов / систем?
  *Пример H2: "Ведическая нумерология и другие системы — в чём разница"*

- Для кого подходит / кому будет полезно?
  *Пример H2: "Кому подойдёт ведическая нумерология"*

- Какие особые случаи и нюансы существуют?
  *Пример H2: "Особые числа и исключения в ведической системе"*

- Распространённые заблуждения / честно о границах?
  *Пример H2: "Что ведическая нумерология может показать — и чего не может"*

- Что спрашивают чаще всего?
  *Пример H2: "Часто задаваемые вопросы"* — **обязателен для всех standalone**

### Lead после H2

Каждый H2 открывается lead-абзацем:
- 3-4 предложения, 70-90 слов строго
- Создаёт образ, задействует воображение
- Числа как живые сущности
- Чередование коротких и длинных предложений

### FAQ-блок

Вопросы берутся из семантики кластера.
Вопросы переформулируй в живой язык — не копируй поисковый запрос дословно.
Ответы: 2-4 предложения, конкретно и по существу.
Никаких примеров с арифметикой в ответах — только принцип словами + CTA на бот.

### Внутренняя перелинковка

**Ссылка вверх (на ближайший hub) — обязательно:**
В конце статьи или в первом абзаце.

**Горизонтальные ссылки — не более 3:**
Приоритет: ближайший hub + 1-2 тематически близких страницы.
Размещай в тексте органично, не списком.

### CTA-блоки

```
[CTA_BLOCK: тип=баннер, место=после_введения]
[CTA_BLOCK: тип=баннер, место=конец_статьи]
```

Не пиши текст CTA — только placeholder.

### Распределение ключевых слов в standalone

| Место | Что размещать |
|-------|--------------|
| Title | Главный ключ |
| Description | Главный ключ + смежные |
| H1 | Главный ключ (живее) |
| Первый абзац | Главный ключ |
| H2 заголовки | Средне- и низкочастотные ключи |
| H3 заголовки | LSI-ключи |
| Тело текста | Все ключи равномерно |
| FAQ вопросы | Вопросные запросы из семантики |
