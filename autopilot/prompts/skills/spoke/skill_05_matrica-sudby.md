## Что делает эту серию уникальной

Матрица судьбы (метод Александрова, также известна как «квадрат Пифагора») —
9 ячеек, каждая = одна цифра 1-9, значение ячейки = сколько раз эта цифра
встретилась в наборе (дата + 4 производных числа).

**Серия — 9 spoke-страниц, по одной на цифру.** Восемь «линий» (группы
из трёх ячеек) НЕ получают отдельных URL — решение принято осознанно:
у линий недостаточно собственного содержания на полноценную статью
(только название + 1-2 предложения силы/слабости, в отличие от ячеек,
где на каждую — 7 уровней глубины, планета, стихия, вопрос). Линии
присутствуют в системе как:
- врезка на странице каждой ячейки — «эта цифра также часть линии X»
- отдельный раздел на самом хабе `/matrica-sudby/`, где перечислены все 8

**Источник данных — не выдумывать, а брать готовое.** Реальные трактовки
(название/домен/7 уровней глубины/планета/стихия/вопрос/практики) взяты
из датасета бота. Файл должен лежать в `content/data/pythagorean-matrix.json`
(копия из бота) — агент берёт формулировки **оттуда**, не сочиняет заново.
Это тот же принцип, что и с `destiny.json` для чисел судьбы.

**Мастер-числа не используются** (`master_numbers: false` в датасете) —
в отличие от чисел судьбы, здесь нет отдельной ветки для 11/22/33.

## Методика расчёта (для вводных абзацев и H2 "Как считается")

**Рабочие числа** — четыре производных числа от даты ДД.ММ.ГГГГ:

```
A = сумма всех цифр даты                                — первичная энергия
B = сумма цифр числа A                                  — скрытый потенциал
C = A минус удвоенная первая НЕНУЛЕВАЯ цифра дня         — энергия преобразования
D = сумма цифр числа C                                  — результирующая энергия цикла
```

⚠️ Важно для точности: «первая цифра дня» — это первая **ненулевая** цифра,
не первый символ. Для дня «01» это 1, а не 0. Эта деталь однажды уже стоила
сайту неверного расчёта на главной странице — при описании методики в
статьях эту тонкость проговаривать не обязательно (см. правило «без
арифметики в тексте»), но сама формула должна быть верной везде, где
считается программно.

Все цифры (дата + A + B + C + D) объединяются в набор, ноль в подсчёт не
идёт — своей ячейки у него нет. Считается, сколько раз встретилась каждая
цифра 1-9 — это и есть значение соответствующей ячейки.

**Сетка** (реальное расположение, сверху вниз):
```
7  8  9   ← Дух      (Везение, Долг, Память)
4  5  6   ← Душа      (Здоровье, Логика, Труд)
1  2  3   ← Тело      (Воля, Энергетика, Познание)
```

  50-70 слов). depth[6] покрывает «6 и более».

## Структура H2

### Обязательные H2:

- **Что означает цифра N в матрице судьбы** — использовать `question` из
  датасета как эмоциональный крючок лида
- **Семь уровней цифры N** — табличный обзор всех значений от 0 до 6+
- **Если цифры N нет в матрице** — полный текст `depth[0].desc`
- **Как усилить цифру N** — практики (`practices.for_zeros`, адаптировать
  под конкретную ячейку)
- **Часто задаваемые вопросы**

### Вариативные H2 (2-3 на выбор):

| H2 | Когда использовать |
|---|---|
| Если цифра N в избытке | все страницы — использовать depth[5]/depth[6] |
| N и здоровье/работа/деньги | по домену конкретной ячейки |
| N у детей рождённых после 2000 | 1, 2 — из-за структуры года 20XX эти ячейки чаще пустые у молодых читателей (математическая особенность метода) |
| Какие линии включают цифру N | опционально, если врезки недостаточно и тема заслуживает отдельного раздела |

Выбирай H2, которые органичны конкретной цифре — не добавляй ради объёма.
Данных с бота достаточно, чтобы набрать 1200-1500 слов почти без
дополнительного вымысла.

## Блок "Карточка ячейки"

**Место:** сразу после lead первого H2.
**Тип блока:** `fact_row`

```json
{

  "items": [
    {"label": "Планета", "value": "Солнце"},

    {"label": "Домен", "value": "Характер, целеустремлённость, самодисциплина"}

}
```

**Место:** в разделе H2 "Семь уровней цифры N".
**Тип блока:** `table`

## Раздел на хабе про линии (для будущей статьи `/matrica-sudby/`)

Хаб должен содержать отдельный H2 (например, «Восемь линий матрицы
судьбы»), где перечислены все 8 линий из таблицы выше — название, клетки,
тема, без 7-уровневой глубины (её там и нет в датасете). Также на хабе
уместен раздел «синтез» — паттерны чтения всей матрицы целиком
(`synthesis.dominant_pattern` из датасета: один доминант / баланс / много
пустых / все ячейки заполнены) — это данные уровня всей матрицы, не
отдельной цифры, поэтому их место на хабе, а не на spoke-страницах.
