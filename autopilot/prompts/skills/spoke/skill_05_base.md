# skill_05_base.md — Базовые правила spoke-страницы

## Что такое spoke-страница

Spoke — узкая глубокая страница которая:
- Раскрывает один конкретный элемент темы
- Ранжируется по узким низко- и среднечастотным запросам
- Всегда связана с родительским hub
- Пишется по шаблону — агент генерирует автоматически

Конкретная структура, уникальные блоки и JSON-пример —
в файле серии. Поле `template` в JSON карточки указывает какой файл читать:

| template | файл серии |
|---|---|
| `chislo-sudby-1-9` | skill_05_chislo-sudby-1-9.md |
| `chislo-sudby-mastery` | skill_05_chislo-sudby-mastery.md |
| `sovmestimost` | skill_05_sovmestimost.md |
| `chasy-00-23` | skill_05_chasy-00-23.md |
| `angelskie-chisla` | skill_05_angelskie-chisla.md |
| `mesyacy` | skill_05_mesyacy.md |
| `matrica-sudby` | skill_05_matrica-sudby.md |

Прочитай этот файл первым — он содержит правила общие для всех серий.
Затем читай файл своей серии — он содержит структуру и JSON-пример.

## Технические параметры

**Объём:** 1200–1500 слов
**Заголовков H2:** 4–6
**Заголовков H3:** 2-3 под каждым H2
**Списков:** 1-2 на статью
**FAQ:** 2-3 вопроса

---

## Бюджет слов — ориентир для объёма 1350

| Часть | Кол-во | Слов за единицу | Итого |
|---|---|---|---|
| Вводный абзац | 1 | 60–80 | ~70 |
| Lead после H2 | 4-5 | 70–90 | ~360 |
| Абзацы в H3 | ~9-10 (2 на H3, ~4-5 H3) | 60–80 | ~700 |
| FAQ (вопрос+ответ) | 2-3 | 50–70 | ~150 |
| fact / quote / list / callout / fact_row | 3-4 | 15–35 | ~90 |

Сумма ≈ 1370 — середина диапазона 1200–1500.
Считай по верху диапазонов, не по минимуму — на коротких spoke-страницах
недобор объёма особенно заметен из-за малого числа блоков.

## Мета-данные

**Title (от 50 до 65 символов):**
- Конкретное число/время/тема + что узнает читатель
- Пример: "Число судьбы 7: значение и характер"
- Пример: "22:22 на часах — значение в нумерологии"

**Description (от 140 до 160 символов):**
- Конкретный ответ на запрос + призыв читать дальше
- Пример: "Число судьбы 7 — знак мудреца и искателя.
  Характер, любовь, карьера и совместимость
  людей с числом 7. Читайте полную расшифровку."

**H1 (от 50 до 65 символов):**
- Живее чем title, но близко к запросу
- Пример: "Число судьбы 7 — путь мудреца"
- Пример: "22:22 на часах: что говорят числа"

---

## Вводный абзац

- 3-4 предложения, 60-80 слов
- Эмоциональный крючок — образ, ощущение, парадокс
- Без шаблонных фраз: "В этой статье мы расскажем...", "Нумерология — это древняя наука..."
- Главный ключ органично в первом-втором предложении

---

## Lead после H2

- 3-4 предложения, строго 70-90 слов — проверяй фактическое число
- Создаёт образ, задействует воображение
- Числа как живые сущности
- Чередование коротких и длинных предложений
- Не пересказывает H2, а разворачивает его смысл

---

## Абзацы в H3

- Два абзаца на каждый H3, 60-80 слов каждый
- Между абзацами — fact или quote если есть реально сильный материал
  (не чаще чем в каждом втором H3, не вставляй ради структуры)
- После второго абзаца — опционально callout, table или list

---

## FAQ

- 2-3 вопроса из семантики кластера
- Вопросы переформулировать в живой язык (правила в skill_03)
- Ответы: 2-4 предложения, конкретно и по существу
- Никаких примеров с арифметикой в ответах

---

## CTA-блоки

Три обязательных CTA на каждой spoke-странице:

**CTA 1 (after_intro)** — сразу после вводного абзаца, ссылка на бота.
**CTA 2 (mid_article_channel)** — после второго H2 (не после уникальных блоков), ссылка на канал. Между CTA 1 и CTA 2 должен быть полноценный раздел с текстом.
**CTA 3 (end_of_article)** — после блока links, ссылка на бота. Другая формулировка чем CTA 1.

Правила:
- `heading` CTA 1 и CTA 3 — разные формулировки, оба про расчёт в боте
- `heading` CTA 2 — про контент канала, не про расчёт
- Все тексты релевантны теме конкретной статьи
- Тексты CTA берёшь из Раздела 5 карточки (промпт 2)

```json
{
  "type": "cta",
  "position": "after_intro | mid_article_channel | end_of_article",
  "heading": "конкретный заголовок под тему статьи",
  "subtext": "одно предложение — усиливает заголовок",
  "button_text": "текст кнопки →",
  "url": "https://t.me/numerolog_master_bot или https://t.me/chisla_vlasti"
}
```

## Внутренняя перелинковка

Все ссылки оформляются единым блоком `links` перед финальным CTA.
Берутся из поля `internal_links` карточки.

**up** — ссылка на родительский hub. Всегда первая в блоке.
**horizontal** — 2-3 ссылки на соседние spoke того же hub.
**down** — 1-2 ссылки на тематически связанные hub.

```json
{
  "type": "links",
  "title": "По теме",
  "items": [
    {"url": "/hub-url/", "anchor": "название hub", "description": "короткое описание"},
    {"url": "/spoke-url/", "anchor": "название spoke", "description": "короткое описание"}
  ]
}
```

---

## Обязательный порядок блоков в конце статьи

```
[faq]
[links]
[cta end_of_article]
```

Этот порядок одинаков для всех серий. Не менять.
