# skill_06_checklist.md

## Финальный чек-лист качества

### Использование
Запускай этот чек-лист после написания
любой страницы (hub, spoke, standalone).
Проверяй каждый пункт — не пропускай.
Если пункт не выполнен — исправь до отправки.

---

## БЛОК 1 — Технические параметры

### Мета-данные
☑ Title написан (до 60 символов)
☑ Title содержит главный ключевой запрос
☑ Title не заканчивается на многоточие
☑ Description написан (до 155 символов)
☑ Description содержит главный ключевой запрос
☑ Description содержит призыв к действию
☑ H1 написан (до 60 символов)
☑ H1 не копирует Title дословно
☑ H1 содержит главный ключевой запрос

### Объём
☑ Hub: 2000–2500 слов
☑ Spoke: 1200–1500 слов
☑ Standalone: 1500–2000 слов
☑ Объём **посчитан**, а не оценён на глаз (см. "Как считать объём" ниже)
☑ Если факт ниже нижней границы — H3-абзацы с самыми короткими описаниями
  дополнены до 90-110 слов (hub) / 70-90 (standalone) / 60-80 (spoke).
  Новые разделы не добавляются, тема не размывается — просто абзацы
  раскрываются полнее там, где они короче нормы.

### Как считать объём

После того как статья написана:
1. Сложи слова во всех paragraph, lead, faq (вопрос+ответ), links, а также
   в text/caption/rows/items спецблоков (fact, quote, table, list, callout, fact_row).
2. Если сумма ниже нижней границы диапазона для типа страницы —
   не публикуй с пометкой "объём в норме" без проверки.
3. Самая частая причина недобора — abзацы после H3 короче 90 слов (hub) /
   70 слов (standalone) / 60 слов (spoke) и lead короче 70 слов.
   Проверь именно их в первую очередь.

### Структура заголовков
☑ Один H1 на странице
☑ Hub: 7–8 заголовков H2, 2–3 H3 под каждым
☑ Spoke: 4–6 заголовков H2, 2 H3 под каждым
☑ Standalone: 6–7 заголовков H2, 2 H3 под каждым
☑ Заголовки не содержат знаков препинания
  (точки, восклицательные, вопросительные)
☑ Заголовки естественные — не SEO-шные

## БЛОК 2 — SEO параметры

### Ключевые слова
☑ Главный ключ есть в первом абзаце
☑ Главный ключ есть в H1
☑ Главный ключ есть минимум в одном H2
☑ Главный ключ есть в заключении
☑ Плотность ключей: 2-3% (без переспама)
☑ Ключи вписаны естественно
  (допустимы слова между частями ключа)
☑ LSI-ключи распределены равномерно
☑ Каждый LSI-ключ использован 1-2 раза
☑ LSI-ключи взяты из семантики кластера или прямых синонимов —
  не добавлены темы которых нет в cluster_keywords

### FAQ-блок
☑ Hub: 3-5 вопросов
☑ Spoke: 2-3 вопроса
☑ Standalone: 2-3 вопроса
☑ Вопросы взяты из реальной семантики кластера
☑ Вопросы переформулированы в живой язык — не скопированы дословно из запросов
☑ Ответы краткие и по существу (2-4 предложения)

### Внутренняя перелинковка
☑ Hub → все дочерние spoke: ссылки в разделе "Изучите подробнее"
☑ Hub → 2-3 соседних hub: ссылки в тексте
☑ Spoke → родительский hub: ссылка обязательна
☑ Spoke → 2-3 соседних spoke: ссылки в тексте
☑ Spoke → 1-2 hub другой темы: ссылки в тексте
☑ Standalone → ближайший hub + 1-2 тематически близких страницы (не более 3 горизонтальных)
☑ Все анкоры естественные с ключевыми словами
☑ Нет битых ссылок (все URL из кластеризации)

Эти правила выполняются здесь, в промпте — вручную, при написании статьи.
После этого шага статья проходит через `autopilot/linkbuilder.js` — он
не заменяет эту работу, а лишь дополнительно проставляет ссылки в те
H2-разделы, где по итогам этого чек-листа не оказалось ни одной ссылки
(см. AUTOPILOT.md).

## БЛОК 3 — Контент качество

### TOV — мистический экспертный
☑ Lead-абзацы после H2 — эмоциональные, 3-4 предложения, 70-90 слов
☑ Абзацы после H3 — информационно-практические
☑ Тон уверенный, без заискивания
☑ Нет менторства ("вы должны", "вам нужно")
☑ Обращение "вы" (не "ты")
☑ Числа описаны как живые сущности
  (не просто математические объекты)

### Расчёты и арифметика
☑ Нет пошаговых примеров с конкретными числами:
  ❌ "складываете 1 и 4, получаете 5"
  ❌ "1+5+0+3 = 9"
  ❌ "если родились 14-го — это 1 и 4"
☑ Алгоритм объяснён принципом словами без вычислений
☑ В разделах где упоминается расчёт — есть CTA на Telegram-бот

### AI-клише — проверь что НЕТ
☑ Нет: "включает в себя", "предлагает"
☑ Нет: "славится", "идеальные условия"
☑ Нет: "насладиться", "преимущества"
☑ Нет: "максимально", "абсолютно", "очень"
☑ Нет: "на самом деле", "в целом", "в общем"
☑ Нет: "осуществить", "произвести расчёт"
☑ Нет: "на этой странице", "кликните здесь"
☑ Нет: "уникальный", "особенный", "удивительный"
☑ Нет: "древняя мудрость", "тайные знания"
☑ Нет: "откройте для себя", "погрузитесь в мир"
☑ Нет шаблонных вступлений:
  "В этой статье мы расскажем..."
  "Нумерология — это древняя наука..."

### E-E-A-T
☑ Есть конкретные примеры из жизни
☑ Есть экспертные объяснения "почему"
☑ Упомянуты нумерологические системы по названию
  (Пифагорейская, Халдейская, Ведическая)
☑ Нет непроверенных утверждений
☑ Нет обещаний которые нумерология не даёт
☑ Честно о границах системы

### Сущности
☑ Конкретика вместо абстракций
☑ Полные названия при первом упоминании
☑ Показаны связи между понятиями
☑ Главная тема доминирует на всей странице

### Форматирование
☑ Абзацы: 3-5 предложений, но не короче целевого числа слов
  из skill_04/04b/05 (90-110 для hub, 70-90 для standalone, 60-80 для spoke) —
  число предложений само по себе не гарантирует нужный объём
☑ Чередование коротких и длинных предложений
☑ Нет стены текста без разбивки

### Блоки контента (skill_07)
☑ Минимум 1 блок fact на статью
☑ Минимум 1 блок quote на статью
☑ Минимум 1 блок table на статью
☑ Минимум 1 блок list на статью
☑ fact_row — ровно 1 раз в первом разделе H2
☑ fact_row в читаемой части — строкой через разделитель, не таблицей
☑ callout — не чаще 1 раза на 2 раздела H2
☑ fact и quote не используются оба в одном разделе H3
☑ table и list не стоят в одном разделе H3
☑ Специальные блоки не идут подряд без параграфа между ними
☑ Разбивка H3 на два абзаца с элементом внимания —
  применяется не чаще чем в каждом втором H3,
  только если есть реально сильный факт или формулировка

## БЛОК 4 — Уникальность

### Нет дублирования
☑ Содержание не копирует другие страницы сайта
☑ Каждое число описано уникально
  (число 7 ≠ число 4 по структуре подачи)
☑ Нет повторения одной мысли разными словами
☑ Если тема есть на другой странице —
  только краткое упоминание + ссылка

### Специфика темы
☑ Страница раскрывает именно свою тему
☑ Нет ухода в смежные темы без возврата
☑ Проверка: если убрать главную тему —
  статья теряет смысл? (должна терять)

## БЛОК 5 — CTA и JSON

### CTA-блоки
☑ Placeholder после введения:
  [CTA_BLOCK: тип=баннер, место=после_введения]
☑ Placeholder в конце статьи:
  [CTA_BLOCK: тип=баннер, место=конец_статьи]
☑ Текст CTA не написан (только placeholder)

### JSON для агента
☑ JSON сформирован корректно
☑ Все поля заполнены
☑ URL соответствует кластеризации
☑ cluster_keywords содержит все запросы кластера
☑ blocks заполнен реальным контентом (не описаниями)
☑ blocks содержит все обязательные типы блоков
☑ Порядок блоков соответствует структуре статьи
☑ internal_links заполнены корректными URL
☑ status: "done" после написания статьи
☑ word_count_target соответствует типу страницы:
  hub — 2500, standalone — 2000, spoke — 1200

## БЛОК 6 — Финальный отчёт

После каждой страницы выведи:

```
## Отчёт по странице

**URL:** /url/
**Тип:** hub / spoke / standalone
**Объём:** XXX слов

**Использованные ключевые слова:**
- [ключ 1]: X раз
- [ключ 2]: X раз

**LSI-ключи использованы:**
- [lsi 1]: X раз
- [lsi 2]: X раз

**Упомянутые сущности:**
- Числа: [список]
- Системы нумерологии: [список]
- Концепции: [список]

**Внутренние ссылки:**
- Вверх (hub): [url]
- Горизонтально: [url1], [url2]
- Вниз: [url1], [url2]

**Чек-лист пройден:** ДА / НЕТ
**Проблемы:** [если есть — описать]
```

## Быстрая проверка (30 секунд)

Если нет времени на полный чек-лист:

1. Title до 60 символов с главным ключом? ✓
2. H1 есть и отличается от Title? ✓
3. Главный ключ в первом абзаце? ✓
4. Нет AI-клише? ✓
5. TOV мистический экспертный? ✓
6. Нет арифметики в тексте? ✓
7. Ссылка на hub (для spoke)? ✓
8. CTA-placeholder на месте? ✓
9. JSON с полем blocks заполнен? ✓
10. Объём посчитан (не на глаз) и попадает в диапазон? ✓
11. FAQ есть? ✓
12. Есть fact, quote, table, list? ✓
13. Специальные блоки разделены параграфами? ✓
