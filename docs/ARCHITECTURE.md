# Архитектура системы

## Общая схема

```
┌─────────────────────────────────────┐
│         РУЧНАЯ ГЕНЕРАЦИЯ             │
│   Промпт 2 → Промпт 3 (чат с Claude) │
│         → content/pending/           │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│           ПУБЛИКАТОР                 │
│      autopilot/publish.js            │
│  pending → published, проставляет    │
│  даты, обновляет planned-urls.json   │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│           ЛИНКБИЛДЕР                 │
│      autopilot/linkbuilder.js        │
│  отдельный ручной шаг — не входит    │
│  в publish.js автоматически          │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│               САЙТ                   │
│   Next.js читает content/published/  │
│      в момент запроса страницы       │
└───────────────────────────────────────┘
```

Планировщика (cron), который запускал бы весь цикл сам, пока нет. На VPS
предполагается cron-задание для `publish.js` (см. `DEPLOYMENT.md`) — но
даже там линковка отдельным шагом, не частью автопилота.

## Генерация статьи (ручной процесс)

```
Кластер из semantic_clusters.json
        ↓
   Промпт 2 (в чате с Claude) → JSON карточка
        ↓
   Промпт 3 (в чате с Claude) → готовая статья с текстом
        ↓
   Сохранить вручную в content/pending/{page_id-с-дефисами}.json
```

Автоматической валидации между Промптом 2 и Промптом 3 нет — `validator.js`
не реализован. Проверка качества (чек-лист `skill_06_checklist.md`)
выполняется тем же диалогом с Claude на этапе генерации, не отдельным
скриптом.

## Публикация и линковка (реализованы, работают отдельно друг от друга)

**Публикация** — `autopilot/publish.js`, запускается вручную или по cron.
Берёт N старейших файлов из `content/pending/` (FIFO по времени изменения
файла), проставляет `date_published`/`date_modified`, перекладывает в
`content/published/`, пересобирает `site/content/planned-urls.json`.

**Линковка** — `autopilot/linkbuilder.js`, запускается вручную, отдельной
командой, до или после публикации (работает с файлом на месте). Проставляет
контекстные внутренние ссылки по словарю анкоров.

Оба скрипта проверены end-to-end на реальном прогоне 34 статей — публикация
батчем, публикация по одной, линковка каждого файла, прод-сборка сайта.

## Хранилище

Реализация — JSON-файлы, читаются и пишутся напрямую через `fs` в каждом
скрипте по отдельности (`publish.js`, `update-planned-urls.js`,
`linkbuilder.js`, `build-anchors.js`, роут `app/[...slug]/page.tsx`).
Общего интерфейса хранилища (`storage.js`) нет — см. `STORAGE.md`.

```
/content/
  /pending/      ← сгенерированы вручную, ждут публикации
  /published/    ← опубликованы
  /error/        ← не используется
  /images/
    /hub/
    /standalone/
    /spoke/
```

## Сайт

Next.js читает статьи из `/content/published/`.
Роут `/[...slug]/page.tsx` — динамический (`ƒ` в выводе `next build`):
читает JSON-файл статьи с диска в момент запроса. Новая статья появляется
на сайте сразу после перекладывания в `/published/`, без пересборки.

Исключение — `sitemap.xml`: статический роут (`○` в выводе `next build`),
читает `planned-urls.json` через статический `import`, зафиксированный на
этапе `npm run build`. Новые URL не появляются в sitemap без пересборки
сайта. Проверено в реальном прод-режиме (`npm run build && npm start`):
поведение подтверждено, в `npm run dev` эта проблема не воспроизводится.
