# Архитектура системы

## Общая схема

```
┌─────────────────────────────────────┐
│           ПЛАНИРОВЩИК               │
│        (cron, раз в сутки)          │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│        ОЧЕРЕДЬ КЛАСТЕРОВ            │
│     /content/pending/*.json         │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│         ПУБЛИКАТОР                  │
│   берёт статью → публикует на сайт  │
│   перекладывает в /published/       │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│        УВЕДОМЛЕНИЕ                  │
│     Telegram Bot → канал            │
└─────────────────────────────────────┘
```

## Генерация батча (отдельный процесс)

```
semantic_clusters.md
        ↓
   Промпт 2 (Claude API)
   → JSON карточка
        ↓
   Валидатор (Claude API)
   → ОК / ошибки → повтор
        ↓
   Промпт 3 (Claude API)
   → статья + финальный JSON
        ↓
   /content/pending/
```

## Два независимых процесса

**Генерация** — запускается вручную батчами по 30-50 статей.
Результат складывается в `/content/pending/`.

**Публикация** — запускается автоматически раз в сутки.
Берёт одну статью из `/content/pending/`, публикует, перекладывает в `/content/published/`.

Процессы не зависят друг от друга.
Если API недоступен — публикация идёт из готового запаса.

## Хранилище

Текущая реализация — JSON файлы.
Все операции с хранилищем — через единый интерфейс `storage.js`.
Миграция в PostgreSQL не затрагивает остальной код.

```
/content/
  /pending/      ← сгенерированы, ждут публикации
  /published/    ← опубликованы
  /error/        ← ошибки генерации
  /images/
    /hub/
    /standalone/
    /spoke/
```

## Сайт

Next.js читает статьи из `/content/published/`.
Новая статья появляется на сайте сразу после перекладывания в `/published/`.
Пересборка не нужна — страницы рендерятся динамически.

## Типы страниц

| Тип | Кол-во | Кто пишет |
|-----|--------|-----------|
| Hub | 9 | Вручную |
| Standalone | 13 | Автопилот |
| Spoke | ~160 | Автопилот |

## Внешние зависимости

| Сервис | Назначение | Критичность |
|--------|-----------|-------------|
| Anthropic API | Генерация статей | Только при генерации батча |
| Telegram Bot API | Уведомления | Некритично |
