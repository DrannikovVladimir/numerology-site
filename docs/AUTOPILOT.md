# Автопилот

## Два режима работы

**Генерация батча** — запускается вручную.
Создаёт пул статей и складывает в `/content/pending/`.

**Публикация** — запускается автоматически раз в сутки.
Берёт одну статью из `/content/pending/` и публикует на сайт.

---

## Генерация батча

### Запуск
```bash
node autopilot/generate.js --type spoke --limit 50
node autopilot/generate.js --type standalone --limit 13
```

### Параметры
```
--type    тип страниц: spoke / standalone / all
--limit   максимум статей за запуск
--from    начать с конкретного кластера (опционально)
```

### Алгоритм
```
1. Загрузить список кластеров из semantic_clusters.json
2. Отфильтровать: тип = --type, статус = pending
3. Запустить батч (3 параллельных потока)

Для каждого кластера:
  а. Промпт 2 → сгенерировать JSON карточку
  б. Валидатор → проверить JSON
     - ОК → идём дальше
     - Ошибки → повторить Промпт 2 (макс. 2 попытки)
     - Не исправилось → сохранить в /error/, перейти к следующему
  в. Промпт 3 → сгенерировать статью + финальный JSON
  г. linkbuilder.js → автоматически проставить контекстные ссылки
     - Читает planned-urls.json, берёт только status=true страницы
     - Для каждого paragraph ищет фразы совпадающие с анкорами
     - Проставляет 1-2 ссылки на H2-раздел, один URL не повторяется
  д. Сохранить в /content/pending/
  д. Пауза 2 сек между группами (rate limit)

4. Вывести отчёт
```

### Отчёт после батча
```
✓ Сгенерировано: 48
✗ Ошибок: 2 (кластеры 31, 44)
→ Готово к публикации: 48 статей
```

### Параллельность
```
3 параллельных потока — безопасно для rate limits API
Пауза 2 сек между группами по 3 кластера
50 статей ≈ 30 минут
```

## Публикатор

### Запуск
Автоматически через cron раз в сутки.
```bash
# crontab
0 10 * * * node /autopilot/publish.js
```

### Алгоритм
```
1. Проверить /content/pending/ — есть ли статьи
2. Нет статей → уведомление в Telegram, выход
3. Взять самый старый файл (FIFO — первым пришёл, первым опубликован)
4. Проверить наличие изображения в /content/images/
   - Есть → прикрепить
   - Нет → опубликовать без изображения
5. Записать в /content/published/
6. Обновить sitemap.xml
7. Переложить файл из /pending/ в /published/
8. Обновить статус кластера в semantic_clusters.json
9. Отправить уведомление в Telegram
```

### Уведомление в Telegram
```
✓ Опубликована новая статья

Название: Число судьбы 7 — путь мудреца
URL: /chislo-sudby/7/
Тип: spoke
Осталось в очереди: 47
```

## Обработка ошибок

| Ситуация | Действие |
|----------|----------|
| API недоступен при генерации | Пауза 30 сек, повтор 3 раза, затем skip |
| Валидация не прошла 2 раза | Сохранить в /error/, перейти дальше |
| /pending/ пуст при публикации | Уведомление в Telegram |
| Изображение отсутствует | Публиковать без изображения |

### linkbuilder.js — автоматическая простановка контекстных ссылок
**Приоритет:** высокий
**Место в пайплайне:** между промптом 3 и сохранением в /pending/

Логика:
1. Читает финальный JSON статьи
2. Читает planned-urls.json — берёт только страницы со статусом true
3. Строит словарь анкоров из semantic_clusters.json (primary_keyword + синонимы)
4. Для каждого paragraph в blocks:
   - Ищет фразы совпадающие с анкорами
   - Проверяет что URL не использован в статье ранее
   - Проверяет что в текущем H2-разделе < 2 ссылок
   - Оборачивает фразу в [фраза](url)
5. Сохраняет обновлённый JSON

Файл: autopilot/linkbuilder.js
Зависимости: planned-urls.json, semantic_clusters.json
